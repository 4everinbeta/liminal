## Cross-Phase Integration Findings

### Critical: Missing confetti in primary completion path (HIGH)

**Affected:** FEEDBACK-01, FEEDBACK-02, FEEDBACK-03

`triggerTaskComplete()` from `frontend/lib/confetti.ts` is imported and used in:
- `frontend/components/TaskCard.tsx` ✓ (Board page only)
- `frontend/app/board/page.tsx` ✓

But **never** in `frontend/app/page.tsx`. The main dashboard `handleCompleteTask()` function (line 325) — which handles:
- Focus mode "Complete" button
- Planning mode task list completion
- Swipe-left-to-complete via `SwipeableTaskCard`

...has no confetti call. The entire primary user workflow is silently missing FEEDBACK-01's dopamine hit.

**Fix:** 3-line change in `frontend/app/page.tsx`:
```typescript
import { triggerTaskComplete } from '@/lib/confetti'; // add to imports

// Inside handleCompleteTask, before await updateTask:
triggerTaskComplete();
```

### Medium: restoreTask not wrapped in offline guard

**Affected:** RECOVERY-04

`frontend/lib/api.ts` — `createTask`, `updateTask`, and `deleteTask` all check `if (!getIsOnline())` and enqueue mutations. `restoreTask` (line ~242) does not. If a user deletes a task while offline and tries to immediately undo, the restore call fails silently.

### Low: CapacitySummary hidden in Focus mode

**Affected:** URGENCY-04

`CapacitySummary` renders inside `{!isFocusMode && ...}` — invisible when user is actively completing tasks in Focus mode, which is exactly when remaining capacity matters most.

### Low: UndoBanner/BottomTabBar z-index layering on iOS

**Affected:** Visual layout only

`UndoBanner` uses `fixed bottom-24` (96px). iOS home indicator + BottomTabBar can exceed this on some devices. The banner may partially obscure the tab bar.
