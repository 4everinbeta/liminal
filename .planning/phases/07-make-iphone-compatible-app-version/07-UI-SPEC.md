## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding |
| sm | 8px | Compact element spacing, button padding internal |
| md | 16px | Default card padding, list item padding |
| lg | 24px | Section padding, modal padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **Touch targets:** Minimum 44px for all interactive elements on mobile (iOS HIG requirement). Apply `min-h-[44px] min-w-[44px]` to tappable elements. The complete button on TaskCard currently uses 24px icon — wrap in a 44px tap zone.
- **Bottom tab bar height:** 56px (tab items) + `env(safe-area-inset-bottom)` for home indicator. Total rendered height varies by device: ~56px on older iPhones, ~90px on iPhone 14/15 Pro with Dynamic Island.
- **Bottom safe area padding:** `env(safe-area-inset-bottom)` — applied to the bottom tab bar container and any fixed-bottom elements. The existing `pb-20` in layout.tsx covers ~80px; replace with `pb-[calc(56px+env(safe-area-inset-bottom))]` for precision.
- **Top safe area padding:** `env(safe-area-inset-top)` — applied to the root layout body or a top bar wrapper. Covers notch and Dynamic Island (typically 44–59px).

Source: D-05 (CONTEXT.md) — safe area insets via `env(safe-area-inset-*)` with `viewport-fit=cover`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Notes:
- Font family: Inter — already loaded via `font-sans` in Tailwind config. No change needed.
- Tab bar labels: 12px at weight 400 (standard iOS tab bar label size). This is the only 5th size allowed — reserved for tab bar chrome only.
- No new font sizes beyond the 4 declared above (excluding 12px tab labels). Existing components already use `text-sm` (14px) and `text-base` (16px) consistently.
- Mobile body text must not fall below 16px to prevent iOS auto-zoom on focused inputs. All `<input>` and `<textarea>` must use `text-base` (16px) or larger.

Source: Existing component patterns (TaskCard uses `font-medium text-text`, `text-xs text-muted`), iOS auto-zoom rule.

## Interaction Contracts

### Bottom Tab Bar

- 4 tabs: Dashboard (home icon), Board (grid icon), [center: Plus capture button], (placeholder for future settings — use ellipsis icon, disabled state in this phase).
- Tab order left-to-right: Dashboard | Board | [Plus] | More (disabled)
- Active tab: accent color (`#4F46E5`) icon + 12px label. Inactive tab: `#6B7280` (muted) icon + label.
- Center Plus button: 56px circle, filled accent (`#4F46E5`), white Plus icon (24px). Elevated 8px above bar baseline with `shadow-lg`. Tapping opens `GlobalQuickCapture` modal.
- Tab bar background: white (`#FFFFFF`) with `border-t border-gray-100` separator. `backdrop-blur-md` may be applied if performance allows on older devices.
- Fixed to bottom with `position: fixed; bottom: 0`. Padding bottom equals `env(safe-area-inset-bottom)` so content clears home indicator.

Source: D-03, D-04 (CONTEXT.md).

### Swipe Gestures on TaskCard

- **Swipe left (complete):** Drag threshold 80px. Reveal layer: green fill (`#10B981` / `secondary` token) with white `CheckCircle2` icon (24px). On release past threshold: trigger `triggerTaskComplete()` confetti + call `onComplete`. Card exits with same animation as existing (`opacity: 0, scale: 0.95`).
- **Swipe right (detail/edit):** Drag threshold 80px. Reveal layer: indigo fill (`#4F46E5`) with white `PencilLine` icon (24px). On release past threshold: open edit modal (`EditTaskModal`).
- **Partial swipe (< 80px):** Spring-return to center. No action triggered.
- **Drag animation:** Use Framer Motion `drag="x"` with `dragConstraints` and `dragElastic: 0.1`. Card translates horizontally; reveal layer scales from 0 to 1 opacity as drag progresses.
- **Haptics:** On completion threshold crossed, trigger `Haptics.impact({ style: ImpactStyle.Medium })` via Capacitor Haptics plugin (Claude's discretion — D-11 area). On detail threshold crossed: `Haptics.impact({ style: ImpactStyle.Light })`.

Source: D-06 (CONTEXT.md) "swipe left to complete, swipe right for detail/edit".

### Offline Banner

- Shown when Zustand `isOnline === false`.
- Position: fixed below the top safe area inset, above page content (`top: env(safe-area-inset-top)`).
- Height: 36px. Background: amber-50 (`#FFFBEB`). Text: amber-700 (`#B45309`), 14px, weight 400.
- Copy: "You're offline. Changes will sync when reconnected." (see Copywriting Contract).
- Framer Motion `AnimatePresence` slide-down on mount, slide-up on unmount. Duration 200ms.
- No dismiss button — banner disappears automatically when `isOnline` returns to `true`.

Source: D-10 (CONTEXT.md).

### Viewport and Safe Area

- Root `<html>` or `<body>` must have: `meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover"`.
- CSS variable declarations in `globals.css`:
  ```css
  :root {
    --sat: env(safe-area-inset-top);
    --sab: env(safe-area-inset-bottom);
    --sal: env(safe-area-inset-left);
    --sar: env(safe-area-inset-right);
  }
  ```
- Main content area: `padding-bottom: calc(56px + env(safe-area-inset-bottom))` — ensures no content is hidden behind bottom tab bar.
- Top content (page headers): `padding-top: env(safe-area-inset-top)` where applicable.

Source: D-05 (CONTEXT.md), existing `pb-20 md:pb-0` pattern in `layout.tsx`.

| Element | Copy |
|---------|------|
| Primary CTA (quick capture) | "Capture Task" |
| Tab label: Dashboard | "Today" |
| Tab label: Board | "Board" |
| Tab label: center capture | (icon only — no label per iOS convention for center buttons) |
| Tab label: More | "More" |
| Offline banner | "You're offline. Changes will sync when reconnected." |
| Sync success toast | "Back online. Syncing changes..." |
| Empty state heading (Dashboard, no tasks) | "Nothing due today" |
| Empty state body | "Use the + button to capture your next task." |
| Swipe left hint (first-run only) | "Swipe left to complete" |
| Swipe right hint (first-run only) | "Swipe right to edit" |

Notes:
- Destructive confirmation uses a modal bottom sheet on mobile (not a blocking alert dialog). Two buttons: "Delete" (filled red, 44px) and "Cancel" (ghost, 44px).

- "Capture Task" label replaces the desktop "Quick Capture" label in the modal for mobile context.
- All copy respects the ADHD-safe tone: no shame language, no urgency language in error states, no "You failed to sync" framing.

Source: CONTEXT.md D-10, REQUIREMENTS.md FEEDBACK-04 ("errors display inline with clear recovery actions"), STATE.md ADHD design principle (no shame-inducing copy).

## Mobile-Specific Constraints

These are binding rules that the executor must not deviate from:

1. **No hover states on mobile.** All `hover:` Tailwind variants must be conditional on `md:` breakpoint or use `@media (hover: hover)` in CSS. Touch devices do not have hover.
2. **Input font size minimum 16px.** Any `<input>` or `<textarea>` must use `text-base` (16px) minimum. Smaller font triggers iOS keyboard zoom, breaking the layout.
3. **44px minimum touch targets.** Wrap any icon button smaller than 44px in a `<span>` or `<button>` with explicit `min-h-[44px] min-w-[44px]` and `flex items-center justify-center`.
4. **No fixed sidebars on mobile.** `AppSidebar` is `hidden` on mobile (`hidden md:block`). Bottom tab bar is the only chrome.
5. **Swipe gestures must not conflict with scroll.** `drag="x"` on `SwipeableTaskCard` must use `dragDirectionLock` so vertical scroll is not blocked. Only activate horizontal drag after 10px horizontal movement.
6. **Capacitor static export.** `next.config.js` must use `output: 'export'`. No server components that rely on Node.js runtime APIs. All data fetching is client-side via `lib/api.ts`.
