**Additional established colors (not accent тАФ use as-is):**

**Forbidden:** Do not use `text-red-600` for interrupted/overdue states. Warm orange (#f97316) is the ADHD-safe override for any attention state that is not a true destructive action.

## Interaction Contract

### Undo Banner (30s window)
1. Task completed or deleted тЖТ `lastCompletedTask` / `lastDeletedTask` written to store with timestamp
2. UndoBanner appears bottom-center, above FAB (`bottom-24`)
3. 1-second polling interval checks timestamp against UNDO_WINDOW_MS (30000ms)
4. User clicks UNDO тЖТ optimistic state update тЖТ API call тЖТ `triggerUpdate()`
5. User clicks X or 30s expires тЖТ banner dismisses, store entry cleared
6. Both `lastCompletedTask` and `lastDeletedTask` can be active simultaneously тАФ UndoBanner shows whichever is most recent (current implementation shows completed task priority тАФ preserve)

### Where You Left Off highlight
1. Dashboard loads тЖТ read `previouslyActiveTaskId` from Zustand (persisted via localStorage)
2. If task exists and is not done/deleted тЖТ apply ring class to TaskCard
3. Highlight is static тАФ no popup, no auto-focus, no scroll-to
4. User resumes by clicking the task card (focus mode) or task row (planning mode) тЖТ `setActiveTaskId()` тЖТ `previouslyActiveTaskId` updated
5. If previously active task is completed/deleted тЖТ ring silently skipped

### Interrupted Badge
1. User navigates away from dashboard while a task is `activeTaskId` тЖТ write `interruptedTaskId` to store with timestamp
2. On next dashboard load тЖТ render interrupted badge on matching TaskCard
3. User clicks badge or task card тЖТ `setActiveTaskId(task.id)` тЖТ clear `interruptedTaskId` from store
4. Badge does NOT trigger Pomodoro restart
5. Trigger events: tab close (beforeunload), route change (Next.js router events), page reload
6. NOT triggered by: timer focus/blur, overnight in-progress status, switching between Focus/Plan modes

### Restore (Recently Deleted)
1. Planning mode shows "Recently Deleted (24h)" section when `deletedTasks.length > 0`
2. Each item: line-through text at 60% opacity тЖТ hover lifts to 100% opacity
3. "Restore" button тЖТ `restoreTask(id)` тЖТ `fetchTasks()`
4. No confirmation required for restore (undo is inherently non-destructive)
