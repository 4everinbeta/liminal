The following steps MUST be performed on a macOS machine with Xcode installed. They cannot run in WSL/Linux.

**Part A — Generate the iOS Xcode project (D-09, D-10):**

1. Open terminal on macOS, navigate to the liminal repo root
2. Build the frontend static export first (cap sync needs the webDir to exist):
   ```bash
   cd frontend && npm ci && NEXT_STATIC_EXPORT=true NEXT_DIST_DIR=.next-clean npm run build && cd ..
   ```
3. Generate the iOS project with CocoaPods (CRITICAL — must use --packagemanager flag):
   ```bash
   npx cap add ios --packagemanager CocoaPods
   ```
4. Sync web assets into the iOS project:
   ```bash
   npx cap sync ios
   ```
5. Install CocoaPods dependencies:
   ```bash
   cd ios/App && pod install && cd ../..
   ```
6. Generate Gemfile.lock:
   ```bash
   cd ios && bundle install && cd ..
   ```
7. Verify the workspace exists:
   ```bash
   test -f ios/App/App.xcworkspace/contents.xcworkspacedata && echo "WORKSPACE OK"
   ```
8. Commit the ios/ directory:
   ```bash
   git add ios/
   git status  # Verify ios/ files are staged (NOT empty — gitignore was updated in Plan 01)
   git commit -m "feat(ios): add Capacitor iOS project with CocoaPods + Fastlane config"
   ```

**Part B — Create external prerequisites:**

1. **Create liminal-certificates repo (D-03):**
   - Go to https://github.com/new
   - Name: `liminal-certificates`
   - Visibility: **Private**
   - Do NOT initialize with README, .gitignore, or license (must be empty)
   - Click "Create repository"

2. **Register App ID in Apple Developer Portal:**
   - Go to https://developer.apple.com/account/resources/identifiers/list
   - Click "+" to register a new identifier
   - Select "App IDs" -> "App"
   - Description: `Liminal`
   - Bundle ID: Explicit -> `com.liminal.adhd`
   - Enable capabilities as needed (none required for basic builds)
   - Click "Register"

3. **Create App Store Connect API key:**
   - Go to https://appstoreconnect.apple.com/access/integrations/api
   - Click "Generate API Key"
   - Name: `Liminal CI`
   - Role: `App Manager`
   - Click "Generate"
   - **Download the .p8 file immediately** (Apple only lets you download it once)
   - Note the **Key ID** (shown in the table)
   - Note the **Issuer ID** (shown at the top of the page)
   - Base64-encode the key: `cat ~/Downloads/AuthKey_XXXXXXXX.p8 | base64 | tr -d '\n'`

4. **Generate a Personal Access Token for Match:**
   - Go to https://github.com/settings/tokens
   - Generate new token (classic) with `repo` scope
   - Copy the token value

5. **Generate Match password:**
   ```bash
   openssl rand -base64 32
   ```
   Save this value — it encrypts the certificates in the Match repo.

6. **Initialize Match (run on macOS from ios/ directory):**
   ```bash
   cd ios
   MATCH_GIT_URL=https://github.com/4everinbeta/liminal-certificates.git \
   MATCH_GIT_USERNAME=4everinbeta \
   MATCH_GIT_TOKEN=<your-pat> \
   MATCH_PASSWORD=<your-match-password> \
   bundle exec fastlane match appstore
   ```
   This will generate the provisioning profile and signing certificate and push them (encrypted) to liminal-certificates.

7. **Configure all 7 GitHub Secrets:**
   Go to https://github.com/4everinbeta/liminal/settings/secrets/actions and add:

   | Secret Name | Value |
   |-------------|-------|
   | `MATCH_PASSWORD` | The passphrase from step 5 |
   | `MATCH_GIT_URL` | `https://github.com/4everinbeta/liminal-certificates.git` |
   | `MATCH_GIT_USERNAME` | `4everinbeta` |
   | `MATCH_GIT_TOKEN` | The PAT from step 4 |
   | `APP_STORE_CONNECT_API_KEY_ID` | Key ID from step 3 |
   | `APP_STORE_CONNECT_API_ISSUER_ID` | Issuer ID from step 3 |
   | `APP_STORE_CONNECT_API_KEY_BASE64` | Base64-encoded .p8 from step 3 |
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer macOS -> Apple Developer Portal | Developer authenticates to create API key and register App ID |
| Developer macOS -> liminal-certificates repo | Match pushes encrypted certs via PAT-authenticated HTTPS |
| GitHub Secrets -> macOS runner | 7 secrets injected as env vars during workflow execution |
| macOS runner -> App Store Connect | Runner uploads IPA using API key credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-07 | Information Disclosure | .p8 API key file | mitigate | Key is base64-encoded and stored only as GitHub Secret APP_STORE_CONNECT_API_KEY_BASE64. The .p8 file must NOT be committed. User instructed to delete .p8 after encoding. Verify: `git grep -r "BEGIN.*PRIVATE KEY"` returns empty |
| T-09-08 | Information Disclosure | PAT for cert repo | mitigate | PAT stored as MATCH_GIT_TOKEN secret with minimum repo scope. Not embedded in MATCH_GIT_URL. Matchfile uses git_basic_authorization to keep token out of git clone logs |
| T-09-09 | Tampering | liminal-certificates repo | mitigate | Repo is private. Match encrypts all certs with AES-256 using MATCH_PASSWORD. CI uses readonly:is_ci to prevent cert modification from runners |
| T-09-10 | Denial of Service | Stale API key or PAT | accept | Low risk — API key and PAT are long-lived by default. If revoked, CI fails with clear auth error. Manual rotation documented in user_setup |
| T-09-11 | Elevation of Privilege | Test tag cleanup | mitigate | Instructions explicitly include deleting v0.0.1-test tag after verification to prevent accidental TestFlight distribution of test builds |
</threat_model>
