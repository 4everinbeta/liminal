**File 3 — ios/fastlane/Fastfile:**

Per D-01 (Match), research Pattern 1, and critical finding about `setup_ci`:

```ruby
default_platform(:ios)

platform :ios do
  desc 'Build signed IPA and upload to TestFlight'
  lane :build do
    # CRITICAL: setup_ci creates a temporary keychain on CI runners.
    # Without this, xcodebuild hangs indefinitely waiting for keychain unlock.
    setup_ci if is_ci

    api_key = app_store_connect_api_key(
      key_id:                ENV['APP_STORE_CONNECT_API_KEY_ID'],
      issuer_id:             ENV['APP_STORE_CONNECT_API_ISSUER_ID'],
      key_content:           ENV['APP_STORE_CONNECT_API_KEY_BASE64'],
      is_key_content_base64: true,
      in_house:              false
    )

    match(
      type:     'appstore',
      readonly: is_ci,
      api_key:  api_key
    )

    build_app(
      workspace:     'App/App.xcworkspace',
      scheme:        'App',
      configuration: 'Release',
      export_method: 'app-store',
      output_name:   'Liminal.ipa'
    )

    upload_to_testflight(
      api_key:                           api_key,
      skip_waiting_for_build_processing: true,
      distribute_external:               false,
      notify_external_testers:           false
    )
  end
end
```

Key details:
- `workspace: 'App/App.xcworkspace'` — relative to ios/ directory (where fastlane runs from)
- `scheme: 'App'` — default scheme name from CocoaPods Capacitor template (assumption A1 from research)
- `readonly: is_ci` — prevents cert regeneration in CI (per D-01 + research anti-pattern)
- `setup_ci if is_ci` — prevents CI build freeze (research pitfall 4)
- `is_key_content_base64: true` — handles .p8 newline encoding (research pitfall 5)

Create `.github/workflows/ios.yml` with the following exact content. Per D-07, this is a separate file from ci.yml. Per D-08, it triggers only on v* tags.

```yaml
name: iOS Build & TestFlight

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    name: Build & Upload to TestFlight
    runs-on: macos-15
    timeout-minutes: 45

    steps:
      - uses: actions/checkout@v4

      - uses: maxim-lobanov/setup-xcode@v1
        with:
          xcode-version: latest-stable

      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'
          cache-dependency-path: frontend/package-lock.json

      - name: Install frontend dependencies
        run: npm ci
        working-directory: ./frontend

      - name: Build Next.js static export
        run: npm run build
        working-directory: ./frontend
        env:
          NEXT_STATIC_EXPORT: 'true'
          NEXT_DIST_DIR: .next-clean

      - name: Sync Capacitor iOS
        run: npx cap sync ios

      - name: Cache CocoaPods
        uses: actions/cache@v4
        with:
          path: ios/App/Pods
          key: ${{ runner.os }}-pods-${{ hashFiles('ios/App/Podfile.lock') }}
          restore-keys: |
            ${{ runner.os }}-pods-

      - name: Install CocoaPods dependencies
        run: pod install
        working-directory: ./ios/App

      - uses: ruby/setup-ruby@v1
        with:
          ruby-version: '3.3'
          bundler-cache: true
          working-directory: ./ios

      - name: Build and upload to TestFlight
        run: bundle exec fastlane ios build
        working-directory: ./ios
        env:
          MATCH_PASSWORD: ${{ secrets.MATCH_PASSWORD }}
          MATCH_GIT_URL: ${{ secrets.MATCH_GIT_URL }}
          MATCH_GIT_USERNAME: ${{ secrets.MATCH_GIT_USERNAME }}
          MATCH_GIT_TOKEN: ${{ secrets.MATCH_GIT_TOKEN }}
          APP_STORE_CONNECT_API_KEY_ID: ${{ secrets.APP_STORE_CONNECT_API_KEY_ID }}
          APP_STORE_CONNECT_API_ISSUER_ID: ${{ secrets.APP_STORE_CONNECT_API_ISSUER_ID }}
          APP_STORE_CONNECT_API_KEY_BASE64: ${{ secrets.APP_STORE_CONNECT_API_KEY_BASE64 }}
```

Key details per research:
- Build order is load-bearing: npm build (with NEXT_STATIC_EXPORT=true) -> cap sync -> pod install -> fastlane build
- `cap sync ios` runs from repo root (where capacitor.config.ts lives), NOT from ios/
- `pod install` runs from ios/App/ (where Podfile lives after cap add)
- `ruby/setup-ruby` with `working-directory: ./ios` reads Gemfile from ios/ directory
- `bundler-cache: true` caches gems from Gemfile.lock
- All 7 secrets passed via env block: MATCH_PASSWORD, MATCH_GIT_URL, MATCH_GIT_USERNAME, MATCH_GIT_TOKEN, APP_STORE_CONNECT_API_KEY_ID, APP_STORE_CONNECT_API_ISSUER_ID, APP_STORE_CONNECT_API_KEY_BASE64
- `timeout-minutes: 45` — generous limit for first builds; macOS runner iOS builds typically take 15-25 min
- `macos-15` runner per D-05
- `maxim-lobanov/setup-xcode@v1` with `latest-stable` per D-06

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Secrets -> Runner env | Secrets cross from GitHub vault to ephemeral macOS runner process environment |
| Runner -> Apple APIs | Runner authenticates to App Store Connect and Match cert repo using secrets |
| Committed files -> Public repo | Any value in committed files is visible to all repo viewers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | ios/fastlane/Matchfile | mitigate | All secret values read from ENV vars (MATCH_GIT_URL, MATCH_GIT_USERNAME, MATCH_GIT_TOKEN) — no literals in file. Verify: `grep -r "ghp_\|github_pat_" ios/fastlane/` returns empty |
| T-09-02 | Information Disclosure | ios/fastlane/Fastfile | mitigate | API key content passed via ENV['APP_STORE_CONNECT_API_KEY_BASE64'] with is_key_content_base64:true — no .p8 content in file. Verify: `grep -r "BEGIN.*PRIVATE KEY" ios/` returns empty |
| T-09-03 | Information Disclosure | .github/workflows/ios.yml | mitigate | All 7 secrets use ${{ secrets.* }} syntax — GitHub redacts these from logs automatically. No literal secret values in workflow file |
| T-09-04 | Information Disclosure | CI logs | mitigate | Matchfile uses git_basic_authorization (not embedded URL token) so PAT is not visible in git clone log output |
| T-09-05 | Elevation of Privilege | Fork PR builds | accept | Tag-only trigger (`on: push: tags: ['v*']`) means fork PRs cannot trigger builds. GitHub also does not pass secrets to fork PR workflows by default |
| T-09-06 | Tampering | Match cert repo | mitigate | Match encrypts certs with AES-256 using MATCH_PASSWORD. readonly:is_ci prevents CI from modifying certs. Only local dev with write access can regenerate |
</threat_model>
