## Interaction Contract

### AI Suggestion Card — Inline Pinned (AI-01, AI-02, AI-03)

**Position:** Relative inline element inside `{!isFocusMode && (...)}` Planning mode section, between Toggles Row and Quick Capture (`TaskForm`).

**Visibility rule:** Render only when `!!aiSuggestion` AND `tasks.find(t => t.id === aiSuggestion.suggested_task_id)` resolves to a task. Do not render with blank title.

**Card contents (top to bottom):**
1. Header row: `Wand2` icon in `bg-primary/10` pill + label text "AI Suggestion: Do This Now" (`text-sm text-gray-500 font-semibold`)
2. Task title: `text-lg font-bold text-gray-900` (task title string)
3. Metadata line: due date formatted as relative string via `UrgencyIndicator` or `"Due [date] · [N] min"` (`text-sm text-gray-500`)
4. Reasoning text: AI-generated explanation (`text-sm text-gray-600`)
5. Button row: Accept (left, full flex-1) + Dismiss (right, full flex-1)

**Enter animation:** `initial={{ y: -16, opacity: 0 }}` → `animate={{ y: 0, opacity: 1 }}`, spring `stiffness: 300, damping: 25` — matches EodSummaryToast pattern (source: EodSummaryToast.tsx line 79).

**Exit animation:** `exit={{ y: -16, opacity: 0 }}` — same spring. Wrapped in `<AnimatePresence>`.

**Card class:** `bg-white border border-gray-100 rounded-2xl p-6 mb-4 shadow-sm` — matches card surface convention from `StatsBar.tsx` pills + `EodSummaryToast.tsx`.

### Accept Button (D-05, D-06)

- Label: "Start This Task" (decisive, action-oriented — Claude's discretion; replaces vague "Accept")
- Icon: `Check` size 16 left of label
- Class: `flex-1 min-h-[44px] py-2 bg-primary text-white rounded-lg hover:bg-primary/90 transition-colors font-semibold flex items-center justify-center gap-2`
- On click: (1) `setActiveTaskId(taskId)` optimistically, (2) `handleSwitchToFocus()` immediately — no confirmation, no delay, (3) `sendAiFeedback(taskId, 'accepted')` async (fire-and-forget with error logging)
- Result: card disappears (aiSuggestion → null), user sees Focus mode

### Dismiss Button (D-07, D-08)

- Label: "Not Now" (communicates session-scoped hiding, avoids implying permanent rejection — Claude's discretion)
- Icon: `X` size 16 left of label
- Class: `flex-1 min-h-[44px] py-2 bg-gray-100 text-gray-700 rounded-lg hover:bg-gray-200 transition-colors font-medium flex items-center justify-center gap-2`
- On click: (1) `setAiSuggestion(null)` optimistically, (2) `sendAiFeedback(taskId, 'dismissed')` async (fire-and-forget with error logging)
- Result: card hides for session; re-appears only if 15-min poll returns a new suggestion

### AI Sorting Toggle (D-09, D-10, D-11)

- Unchanged — existing toggle in Planning mode Toggles Row (`sortingMode: 'ai' | 'manual'`)
- No AI score badges or indicators on `PlanningTaskRow` — task order silently reflects `ai_relevance_score` when AI sorting is active
- Label text unchanged: "AI Sorting" (D-11)

## States

| State | Trigger | UI Behaviour |
|-------|---------|-------------|
| Card visible | `aiSuggestion` non-null AND matching task found in `tasks[]` | Inline card renders with enter animation |
| Card hidden (no suggestion) | `aiSuggestion` is null | Card not rendered; no placeholder or skeleton shown |
| Card loading | `getAiSuggestion()` in-flight on page load / poll | Card not shown until response returns; no spinner in card position |
| Accept in-flight | User clicked "Start This Task" | `aiSuggestion` set to null optimistically (card gone), `sendAiFeedback` awaits silently |
| Dismiss in-flight | User clicked "Not Now" | `aiSuggestion` set to null optimistically (card gone), `sendAiFeedback` awaits silently |
| AI sorting active | `sortingMode === 'ai'` | Tasks silently re-ordered by `ai_relevance_score` descending; no per-card indicator |
| AI sorting inactive | `sortingMode === 'manual'` | Task order unchanged; AI card may still appear if suggestion is active |

## Accessibility

- Accept and Dismiss buttons: minimum touch target `min-h-[44px]` (WCAG 2.5.5)

- `Check` icon in Accept button: decorative, `aria-hidden="true"`
- `X` icon in Dismiss button: decorative, `aria-hidden="true"`
- Card container: no `role` override needed (it is a `div` with visible action buttons)
- Focus management: no forced focus on card appearance — user is mid-task-list; card enters passively
