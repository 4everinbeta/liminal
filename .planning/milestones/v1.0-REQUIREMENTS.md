# Requirements Archive: v1.0 ADHD Optimization MVP

**Archived:** 2026-03-28
**Original defined:** 2026-01-31
**Core Value:** ADHD brains respond to NOW/NOT NOW (urgency), not IMPORTANT/NOT IMPORTANT (priority)

---

## v1 Requirements — Final Status

All requirements checked as complete below. See traceability table for per-requirement outcomes.

### Frictionless Capture

- [x] **CAPTURE-01**: User can create task with title only (all other fields optional/auto-filled)
- [x] **CAPTURE-02**: User can add task via voice input (Web Speech API)
- [x] **CAPTURE-03**: User sees task creation confirmation within 100ms (optimistic update)
- [x] **CAPTURE-04**: Quick-add input persists across page navigation (always accessible)
- [x] **CAPTURE-05**: Form data auto-saves every 2 seconds (prevents loss on distraction)
- [x] **CAPTURE-06**: User can capture task in <5 seconds from thought to saved

### Immediate Feedback

- [x] **FEEDBACK-01**: Task completion triggers visual celebration (confetti animation) — *gap fixed in post-audit commit*
- [x] **FEEDBACK-02**: All user actions show immediate visual response (no >200ms lag perception) — *gap fixed in post-audit commit*
- [x] **FEEDBACK-03**: Task check-off has satisfying animation (smooth, rewarding transition) — *gap fixed in post-audit commit*
- [x] **FEEDBACK-04**: Errors display inline with clear recovery actions (no blocking modals)

### Forgiveness & Recovery

- [x] **RECOVERY-01**: User can undo task completion (30 second undo window)
- [x] **RECOVERY-02**: Session state persists across page reload (Zustand persist middleware)
- [x] **RECOVERY-03**: Draft task inputs auto-recover after browser crash
- [x] **RECOVERY-04**: User can restore recently deleted tasks (soft delete with 24hr retention)

### Single Primary Interface

- [x] **INTERFACE-01**: Dashboard presents one primary workflow (not chat + form + board competing)
- [x] **INTERFACE-02**: Secondary interfaces accessible via clear navigation (not all visible at once)
- [x] **INTERFACE-03**: Focus mode is default view (execution over planning)
- [x] **INTERFACE-04**: User can toggle between views without losing context

### Urgency System

- [x] **URGENCY-01**: Tasks change color as deadline approaches (visual time pressure)
- [x] **URGENCY-02**: Tasks show "X hours left" countdown (scarcity framing)
- [x] **URGENCY-03**: Overdue tasks have distinct visual treatment (not shame-inducing)
- [x] **URGENCY-04**: "Due today" section shows time-boxed capacity — *planning mode only (low severity)*
- [x] **URGENCY-05**: Tasks visually "age" after 3 days in backlog (fade/change style)
- [x] **URGENCY-06**: User receives browser notification 1 hour before deadline (opt-in)

### Gamification & Momentum

- [x] **GAMIFY-01**: User sees daily completion count ("3 done today")
- [x] **GAMIFY-02**: User sees weekly streak ("5 days in a row")
- [x] **GAMIFY-03**: User sees concrete impact ("You freed up 2 hours today") — *requires estimated_duration on tasks*
- [x] **GAMIFY-04**: Streak system is flexible (no penalties for breaks, optional participation)
- [x] **GAMIFY-05**: Daily summary shows wins highlighted (no focus on incomplete tasks)
- [x] **GAMIFY-06**: Progress visualizations use animations (smooth, satisfying transitions)

### Voice Capture

- [x] **VOICE-01**: User can press microphone button to start voice input
- [x] **VOICE-02**: Voice input transcribes to task title in real-time
- [x] **VOICE-03**: Voice input works offline (Web Speech API local fallback)
- [x] **VOICE-04**: Voice input has keyboard alternative (accessibility)
- [x] **VOICE-05**: Voice recording has clear visual feedback (pulsing indicator)

### AI Prioritization

- [x] **AI-01**: User sees "Do This Now" AI suggestion at top of task list
- [x] **AI-02**: AI suggestion considers time of day, deadline proximity, estimated duration
- [x] **AI-03**: User can easily dismiss/override AI suggestion (one click)
- [x] **AI-04**: AI adapts to user patterns (learns what gets done vs ignored)
- [x] **AI-05**: AI suggestion updates every 15 minutes (fresh recommendations)
- [x] **AI-06**: Task list auto-sorts by urgency + AI score (manual override available)

### Simplified Scoring

- [x] **SIMPLIFY-01**: Priority/value/effort scores auto-calculated from due date + estimated duration
- [x] **SIMPLIFY-02**: User can optionally tweak scores with shorthand syntax (!high, 30m) — *shorthand only, not full natural language*
- [x] **SIMPLIFY-03**: Numeric 1-100 inputs replaced with presets — *TaskForm complete; EditTaskModal partial*
- [x] **SIMPLIFY-04**: Task form shows only title + due date by default (progressive disclosure)

### Working Memory Support

- [x] **MEMORY-01**: User sees "Currently working on" task prominently displayed
- [x] **MEMORY-02**: User can mark task "paused" and it auto-resumes on next session — *fixed in post-audit commit*
- [x] **MEMORY-03**: Dashboard shows "Where you left off" on page load
- [x] **MEMORY-04**: Interruptions tracked and surfaced ("You were working on X before distraction")

### Anti-Features (Explicitly Removed)

- [x] **REMOVE-01**: Remove multi-field mandatory form (TaskForm requires only title in v1)
- [x] **REMOVE-02**: Remove board drag-drop gating modals (allow incomplete tasks to move)
- [x] **REMOVE-03**: Make Themes/Initiatives optional (not required for basic workflow)
- [x] **REMOVE-04**: Remove "Pause" button stub (replaced with functional pause)
